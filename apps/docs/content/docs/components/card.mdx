# Card

Card component for grouping related content in a visually distinct container. Includes composable header, content, and footer sections with flexible styling options.

## Installation

```bash
npm install forteui-core
```

## Import

```tsx
import { Card, CardHeader, CardContent, CardFooter } from 'forteui-core';
```

## Basic Usage

```tsx
import { Card, CardHeader, CardContent, CardFooter } from 'forteui-core';

function MyComponent() {
  return (
    <Card>
      <CardHeader>
        <h3>Card Title</h3>
      </CardHeader>
      <CardContent>
        <p>This is the card content area.</p>
      </CardContent>
      <CardFooter>
        <button>Action</button>
      </CardFooter>
    </Card>
  );
}
```

## Card Structure

### Card with Header and Content

```tsx
<Card>
  <CardHeader>
    <div>
      <h3 className="text-lg font-semibold">Project Update</h3>
      <p className="text-sm text-gray-500">2 hours ago</p>
    </div>
  </CardHeader>
  <CardContent>
    <p>The new features have been successfully deployed to production. 
       All systems are running smoothly.</p>
  </CardContent>
</Card>
```

### Card with All Sections

```tsx
<Card>
  <CardHeader>
    <div className="flex items-center justify-between">
      <h3 className="text-lg font-semibold">Team Settings</h3>
      <Badge variant="soft" color="primary">Pro</Badge>
    </div>
  </CardHeader>
  <CardContent>
    <div className="space-y-4">
      <div>
        <label className="text-sm font-medium">Team Name</label>
        <Input defaultValue="Design Team" />
      </div>
      <div>
        <label className="text-sm font-medium">Description</label>
        <textarea 
          className="w-full p-2 border rounded" 
          placeholder="Describe your team..."

      </div>
    </div>
  </CardContent>
  <CardFooter className="flex justify-between">
    <Button variant="outline-default">Cancel</Button>
    <Button variant="primary">Save Changes</Button>

```

## Design Specifications

- **Border Radius**: 8px for modern appearance
- **Shadow**: Subtle elevation for depth
- **Border**: 1px solid border for definition
- **Padding**: Consistent spacing between sections
- **Background**: White background with proper contrast

## Accessibility

- **Semantic Structure**: Uses proper heading hierarchy
- **Focus Management**: Interactive cards support keyboard navigation
- **Screen Readers**: Content is properly structured and announced
- **Color Contrast**: Maintains WCAG AA compliance

## Best Practices

1. **Consistent spacing**: Use consistent padding and margins
2. **Clear hierarchy**: Use proper heading levels and visual hierarchy
3. **Logical grouping**: Group related content together
4. **Action placement**: Place primary actions in footer or prominent locations
5. **Responsive design**: Ensure cards work well on all screen sizes
6. **Loading states**: Consider loading states for dynamic content
