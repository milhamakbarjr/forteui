# Radio

Radio component for single-choice selection controls. Perfect for forms where users need to select one option from a set of mutually exclusive choices.

## Installation

```bash
npm install forteui-core
```

## Import

```tsx
import { Radio, RadioGroup } from 'forteui-core';
```

## Basic Usage

```tsx
import { Radio, RadioGroup } from 'forteui-core';

function MyComponent() {
  return (
    <RadioGroup defaultValue="option1">
      <Radio value="option1" label="Option 1" />
      <Radio value="option2" label="Option 2" />
      <Radio value="option3" label="Option 3" />
    </RadioGroup>
  );
}
```

## Controlled RadioGroup

```tsx
import { useState } from 'react';

function ControlledRadioExample() {
  const [value, setValue] = useState('option2');
  
  return (
    <RadioGroup value={value} onValueChange={setValue}>
      <Radio value="option1" label="Option 1" />
      <Radio value="option2" label="Option 2" />
      <Radio value="option3" label="Option 3" />
    </RadioGroup>
  );
}
```

## Props

### RadioGroup Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Controlled selected value |
| `defaultValue` | `string` | - | Uncontrolled default value |
| `onValueChange` | `(value: string) => void` | - | Callback when selection changes |
| `disabled` | `boolean` | `false` | Disable the entire group |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Radio components |

### Radio Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Value for this radio option |
| `label` | `string` | - | Label text |
| `variant` | `'default' \| 'primary' \| 'secondary' \| 'info' \| 'success' \| 'warning' \| 'error'` | `'default'` | Visual variant |
| `size` | `'sm' \| 'md'` | `'md'` | Size of the radio |
| `disabled` | `boolean` | `false` | Whether this option is disabled |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Additional content below label |

## Design Specifications

- **Size**: Small (16px), Medium (20px)
- **Shape**: Circular with inner dot
- **Border**: 2px solid for clear definition
- **Focus Ring**: 2px outline for accessibility
- **Selected State**: Filled inner circle
- **Animations**: Smooth transitions for state changes

## Accessibility

- **Keyboard Navigation**: Arrow keys navigate between options
- **Screen Readers**: Properly announced with group context
- **Focus Management**: Clear focus indicators
- **ARIA**: Follows ARIA radio group specifications
- **Label Association**: Labels properly associated with controls

## Best Practices

1. **Mutually exclusive**: Use for options where only one can be selected
2. **Clear labels**: Use descriptive, concise labels
3. **Logical order**: Arrange options in logical sequence
4. **Default selection**: Consider providing a sensible default
5. **Group related options**: Keep related choices together
