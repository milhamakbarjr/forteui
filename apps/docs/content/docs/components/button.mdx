# Button

A versatile button component that supports multiple variants, sizes, states, and rich icon integration. Built with accessibility in mind and following the ForteUI design specifications.

## Installation

The Button component is included in the ForteUI core package.

```bash
npm install forteui-core
```

## Import

```tsx
import { Button } from 'forteui-core';
```

## Basic Usage

The simplest way to use the Button component:

```tsx
import { Button } from 'forteui-core';

function MyComponent() {
  return (
    <Button variant="primary" onClick={() => alert('Clicked!')}>
      Click me
    </Button>
  );
}
```

### Icon Buttons

Icon buttons are circular buttons designed specifically for icon-only actions. They provide hover backgrounds and are perfect for toolbars and compact interfaces.

```tsx
import { Button, IconHome, IconUser, IconPlus, IconX } from 'forteui-core';

// Different icon button variants
<Button variant="icon-default" size="icon-md"><IconHome size={20} /></Button>
<Button variant="icon-primary" size="icon-md"><IconUser size={20} /></Button>
<Button variant="icon-info" size="icon-md"><IconPlus size={20} /></Button>
<Button variant="icon-error" size="icon-md"><IconX size={20} /></Button>
```

#### Icon Button Sizes

Icon buttons come in four different sizes:

```tsx
<Button variant="icon-primary" size="icon-xs"><IconHome size={16} /></Button>
<Button variant="icon-primary" size="icon-sm"><IconHome size={18} /></Button>
<Button variant="icon-primary" size="icon-md"><IconHome size={20} /></Button>
<Button variant="icon-primary" size="icon-lg"><IconHome size={24} /></Button>
```

## Icons

ForteUI includes Tabler Icons out of the box. You can import icons directly from the core package.

### Import Icons

```tsx
import { Button, IconPlus, IconCheck, IconTrash, IconHome, IconUser } from 'forteui-core';
```

### Buttons with Leading Icons

Add icons before the button text:

```tsx
<Button variant="primary">
  <IconPlus size={16} />
  <span>Add Item</span>
</Button>

<Button variant="success">
  <IconCheck size={16} />
  <span>Confirm</span>
</Button>

<Button variant="error">
  <IconTrash size={16} />
  <span>Delete</span>
</Button>
```

### Buttons with Trailing Icons

Add icons after the button text:

```tsx
import { Button, IconArrowRight, IconChevronDown, IconDownload } from 'forteui-core';

<Button variant="primary">
  <span>Next</span>
  <IconArrowRight size={16} />
</Button>

<Button variant="secondary">
  <span>Options</span>
  <IconChevronDown size={16} />
</Button>

<Button variant="success">
  <span>Download</span>
  <IconDownload size={16} />
</Button>
```

### Icon-Only Buttons

For icon-only buttons, use the icon variants:

```tsx
import { Button, IconHome, IconUser, IconSettings } from 'forteui-core';

<Button variant="icon-default" size="icon-md">
  <IconHome className="w-6 h-6" />
</Button>

<Button variant="icon-primary" size="icon-md">
  <IconUser className="w-6 h-6" />
</Button>

<Button variant="icon-info" size="icon-md">
  <IconSettings className="w-6 h-6" />
</Button>
```

#### Icon Button Sizes

Icon buttons come in four sizes:

## Advanced Usage

### As Child (Polymorphic)

Use the `asChild` prop to render the button as a different element:

```tsx
import { Button } from 'forteui-core';
import Link from 'next/link';

<Button asChild>
  <Link href="/dashboard">Go to Dashboard</Link>
</Button>
```

### Custom Click Handlers

```tsx
const handleClick = () => {
  console.log('Button clicked!');
  // Your custom logic here
};

<Button variant="primary" onClick={handleClick}>
  Custom Handler
</Button>
```

## API Reference

### Button Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `ButtonVariant` | `'primary'` | Visual style variant |
| `size` | `ButtonSize` | `'md'` | Button size |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `asChild` | `boolean` | `false` | Render as a child element (polymorphic) |
| `onClick` | `(event: MouseEvent) => void` | - | Click handler function |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Button content |

## Design Specifications

### Contained Buttons
- **Height**: Small (30px), Medium (36px), Large (48px)
- **Border Radius**: 6px
- **Font**: Plus Jakarta Sans Bold
- **Font Size**: 13px (sm), 14px (md), 15px (lg)
- **Hover State**: Darker shade of the primary color
- **Focus Ring**: 2px solid primary color

### Icon Buttons
- **Sizes**: XS (28px), Small (36px), Medium (40px), Large (48px)
- **Shape**: Circular
- **Icon Size**: Scales proportionally with button size
- **Hover State**: 8% opacity background color

## Accessibility

- **Keyboard Navigation**: Supports Tab and Enter/Space key interaction
- **Screen Readers**: Properly labeled with accessible text
- **Focus Management**: Clear focus indicators for keyboard navigation
- **Disabled State**: Properly communicated to assistive technologies
- **ARIA**: Follows ARIA button specifications

## Best Practices

1. **Use appropriate variants**: Primary for main actions, secondary for supporting actions
2. **Consistent sizing**: Use the same size for buttons in a group
3. **Clear labels**: Use descriptive text that clearly indicates the action
4. **Icon alignment**: Ensure icons are properly sized and aligned with text
5. **Loading states**: Consider adding loading indicators for async actions
