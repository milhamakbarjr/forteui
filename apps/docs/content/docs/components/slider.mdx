# Slider

A versatile range input component for selecting numeric values within a specified range. Supports single and dual handle modes, step controls, custom formatting, and accessibility features.

## Installation

```bash
npm install forteui-core
```

## Import

```tsx
import { Slider } from 'forteui-core';
```

## Basic Usage

```tsx
import { Slider } from 'forteui-core';

function MyComponent() {
  return (
    <Slider 
      min={0}
      max={100}
      defaultValue={50}
      label="Volume"
    />
  );
}
```

## Single Handle Slider

### Basic Single Slider

```tsx
import { useState } from 'react';

function SingleSliderExample() {
  const [value, setValue] = useState(25);
  
  return (
    <div className="space-y-6">
      <Slider 
        label="Temperature"
        min={0}
        max={100}
        value={value}
        onChange={setValue}
        suffix="°C"
      />
      
      <p className="text-sm text-gray-600">
        Current temperature: {value}°C
      </p>
    </div>
  );
}
```

## Controlled vs Uncontrolled

### Controlled Slider

```tsx
import { useState } from 'react';

function ControlledSlider() {
  const [value, setValue] = useState(50);
  
  return (
    <Slider 
      label="Controlled"
      value={value}
      onChange={setValue}
      showValue
    />
  );
}
```

### Uncontrolled Slider

```tsx
function UncontrolledSlider() {
  return (
    <Slider 
      label="Uncontrolled"
      defaultValue={75}
      showValue
    />
  );
}
```

## Event Handling

### onChange and onChangeEnd

```tsx
import { useState } from 'react';

function EventHandlingExample() {
  const [value, setValue] = useState(50);
  const [endValue, setEndValue] = useState(50);
  
  return (
    <div className="space-y-4">
      <Slider 
        label="Volume"
        value={value}
        onChange={setValue}
        onChangeEnd={setEndValue}
        showValue
      />
      
      <div className="text-sm text-gray-600">
        <p>Current value: {value}</p>
        <p>End value: {endValue}</p>
      </div>
    </div>
  );
}
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `number` | - | Controlled value |
| `defaultValue` | `number` | - | Default uncontrolled value |
| `onChange` | `(value: number) => void` | - | Change handler |
| `onChangeEnd` | `(value: number) => void` | - | Called when dragging ends |
| `min` | `number` | `0` | Minimum value |
| `max` | `number` | `100` | Maximum value |
| `step` | `number` | `1` | Step increment |
| `label` | `string` | - | Label text |
| `showValue` | `boolean` | `false` | Show current value |
| `disabled` | `boolean` | `false` | Disabled state |
| `className` | `string` | - | Additional CSS classes |

## Accessibility

- **Keyboard Navigation**: Arrow keys for fine control, Page Up/Down for large steps, Home/End for min/max
- **Screen Readers**: Proper ARIA labels and value announcements
- **Focus Management**: Clear focus indicators on handles
- **Touch Support**: Touch-friendly handle sizes for mobile devices
- **ARIA Attributes**: Uses `role="slider"` with appropriate value attributes

## Best Practices

1. **Clear Labels**: Always provide descriptive labels
2. **Appropriate Steps**: Choose step values that make sense for your use case
3. **Visual Feedback**: Show current values clearly with `showValue` prop
4. **Performance**: Use `onChangeEnd` for expensive operations
5. **Touch Targets**: Ensure handles are large enough for touch interaction
