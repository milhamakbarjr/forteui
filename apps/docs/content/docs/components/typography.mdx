# Typography

ForteUI's typography system provides a comprehensive set of text components built with Plus Jakarta Sans. Designed for readability, accessibility, and visual hierarchy across all platforms.

## Installation

```bash
npm install forteui-core
```

## Import

```tsx
import { Heading, Text } from 'forteui-core';
```

## Font Family

ForteUI uses **Plus Jakarta Sans** as the primary typeface, providing excellent readability and modern aesthetics across digital interfaces.

```css
font-family: 'Plus Jakarta Sans', -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', sans-serif;
```

## Responsive Typography

### Mobile-First Approach

```tsx
<div className="space-y-4">
  <Heading 
    level={1} 
    className="text-2xl md:text-3xl lg:text-4xl"
  >
    Responsive Heading
  </Heading>
  
  <Text 
    variant="body" 
    className="text-sm md:text-base lg:text-lg"
  >
    This text scales appropriately across different screen sizes, 
    ensuring optimal readability on all devices.
  </Text>
</div>
```

### Breakpoint-Specific Variants

```tsx
<div className="space-y-4">
  {/* Mobile: body-sm, Tablet: body, Desktop: body-lg */}
  <Text className="text-sm md:text-base lg:text-lg">
    Adaptive text size for different viewports
  </Text>
  
  {/* Mobile: h3, Tablet: h2, Desktop: h1 */}
  <Heading className="text-xl md:text-2xl lg:text-3xl" level={1}>
    Scaling headline across devices
  </Heading>
</div>
```

## Props

### Heading Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `level` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | `1` | Heading semantic level |
| `weight` | `'light' \| 'regular' \| 'medium' \| 'semibold' \| 'bold' \| 'extrabold'` | `'semibold'` | Font weight |
| `color` | `'primary' \| 'secondary' \| 'default' \| 'muted' \| 'success' \| 'warning' \| 'error'` | `'default'` | Text color |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Text content |

### Text Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'body-xl' \| 'body-lg' \| 'body' \| 'body-sm' \| 'caption' \| 'overline'` | `'body'` | Text variant |
| `weight` | `'light' \| 'regular' \| 'medium' \| 'semibold' \| 'bold' \| 'extrabold'` | `'regular'` | Font weight |
| `color` | `'primary' \| 'secondary' \| 'default' \| 'muted' \| 'success' \| 'warning' \| 'error'` | `'default'` | Text color |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Text content |

## Accessibility

- **Semantic HTML**: Uses proper heading hierarchy (h1-h6)
- **WCAG Compliance**: All text colors meet AA contrast standards
- **Screen Readers**: Semantic markup ensures proper content navigation
- **Focus Management**: Headings create logical content outline
- **Responsive Text**: Scales appropriately for different devices

## Best Practices

1. **Hierarchy**: Maintain logical heading structure (h1 → h2 → h3)
2. **Contrast**: Ensure sufficient color contrast for accessibility
3. **Readability**: Use appropriate line lengths (45-75 characters)
4. **Consistency**: Apply typography consistently across the application
5. **Performance**: Plus Jakarta Sans is optimized for web performance
6. **Responsive**: Scale typography appropriately for different screen sizes
